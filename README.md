# FTree 家谱管理系统

> 记录家族传承，铭刻世代印记。

FTree 是一款家谱记录与可视化管理桌面软件，支持最多 **5 万人**的家族信息录入、管理和树形关系图渲染。软件以 Electron 桌面应用形式分发，同时支持本地启动后通过浏览器访问。数据采用 JSON 文件本地加密存储，无需安装任何第三方数据库。

## ✨ 功能特性

### 📝 人员信息管理

- 表单化录入家族成员信息
- **必填字段**：姓名、性别、代数（第几代）、父节点（父亲关联）
- **可选字段**：字/号、生卒年月、出生地/籍贯、职业、联系方式（电话/地址）、配偶姓名、信息简介/备注、头像照片
- 支持新增、编辑、删除成员操作
- 支持按姓名、代数搜索和筛选
- 支持子女排行管理和同辈排序

### 🌳 家谱树形图可视化

- 基于 **D3.js** 的交互式树形图，从上至下展示家族关系
- 支持鼠标滚轮缩放、拖拽平移浏览
- 点击节点查看成员详细信息
- 按世代折叠/展开子树，适应大数据量场景
- 右键菜单支持：查看详情、编辑、添加子女、添加配偶、展开后代、删除
- 搜索定位功能：输入姓名快速定位到对应节点并高亮
- 祖先链高亮追踪
- 支持收养关系连线显示
- 多种连线样式可选

### 🎨 个性化显示设置

- **世代配色**：自定义每一世节点的颜色方案，男女可分别配色
- **字辈管理**：管理家族字辈（派语），为每一世设置辈分字，用于取名参考和族谱展示

### 🖼️ 高清导出与打印

- 支持将家谱树导出为高清 **SVG** 图片
- 支持导出全部族谱或指定人物的世系
- 内置打印功能，支持自定义打印布局

### 🔒 数据安全

- 启动密码保护，首次使用设置密码，后续启动需验证
- 数据文件 **AES-256-GCM** 加密存储，密钥由用户密码经 PBKDF2 派生
- 支持修改密码

### 💾 数据管理

- **导出数据**：将所有家族数据导出为明文 JSON 文件，便于备份和迁移
- **导入数据**：从 JSON 文件导入数据
- **自动备份**：每次保存自动保留最近 5 个版本的备份
- **手动备份**：一键创建数据备份
- **清除数据**：支持清除所有数据（需二次确认）

### 📁 自定义数据存储位置

- 数据默认存储在用户主目录下的 `ftree-data/` 文件夹
- 支持自定义数据存储路径（如迁移到外部磁盘或 NAS）
- 修改路径时可选择迁移现有数据
- 一键恢复默认存储位置

### 🌐 双模式访问

- **Electron 桌面应用**：双击 exe 安装后即可使用
- **浏览器访问模式**：本地启动后端服务，局域网内可通过浏览器访问同一界面

## 🚀 快速开始

### 系统要求

- **操作系统**：Windows 10/11（64 位）、macOS、Linux
- **Node.js**：18.0 或更高版本（仅开发时需要）
- **磁盘空间**：约 200MB（安装后）

### 方式一：使用安装包（推荐）

1. 从发布页面下载最新的 Windows 安装包（`.exe`）
2. 双击安装，支持自定义安装目录
3. 安装完成后从桌面快捷方式或开始菜单启动 **FTree**

### 方式二：从源码运行

```bash
# 克隆项目
git clone <仓库地址>
cd FTree

# 安装依赖
npm install

# 开发模式（Electron 桌面应用）
npm run dev:electron

# 开发模式（纯浏览器模式）
npm run dev:server   # 先启动后端服务
npm run dev          # 再启动前端开发服务器
# 然后浏览器访问 http://localhost:5173
```

### 方式三：使用启动脚本（浏览器模式）

```bash
cd FTree

# Windows
start.bat

# 或直接
node start-server.js
# 然后浏览器访问 http://localhost:17800
```

## 📖 使用指南

### 1. 首次使用 — 设置密码

首次启动应用后，会进入密码设置页面：

1. 输入一个至少 **4 位**的密码
2. 确认密码后点击「设置密码」
3. 密码设置成功后自动进入主界面

> ⚠ 请牢记密码！密码用于解密数据文件，遗忘后数据将无法恢复。

### 2. 登录

后续每次启动应用，输入密码即可登录进入主界面。

### 3. 编辑管理 — 录入家族成员

进入「编辑管理」页面（左侧导航第一项）：

**添加第一个成员（始祖）：**
1. 点击右上角「新增成员」按钮
2. 填写姓名、性别、代数填写 `1`，父节点留空（始祖没有父亲）
3. 根据需要填写可选字段（字号、生卒年月、籍贯等）
4. 点击「保存」

**添加后续成员：**
1. 点击「新增成员」
2. 填写姓名、性别、代数
3. 在「父节点」下拉选择器中选择其父亲
4. 完善其他信息后保存

**编辑/删除成员：**
- 在左侧列表中点击选中一个成员，右侧显示其详细信息
- 修改信息后点击「保存」
- 点击「删除」可移除该成员（删除有子女的成员需先处理其子女）

**搜索和筛选：**
- 左侧列表顶部提供搜索框，输入姓名即可搜索
- 可按代数进行筛选

### 4. 家谱图 — 可视化浏览

进入「家谱图」页面（左侧导航第二项）：

**基本操作：**
- **缩放**：鼠标滚轮 或 工具栏的 +/- 按钮
- **平移**：按住鼠标左键拖拽
- **适应屏幕**：点击工具栏的「适应屏幕」按钮

**节点交互：**
- **左键单击节点**：选中节点，显示高亮
- **右键点击节点**：弹出操作菜单
  - 👁 查看详情：打开右侧详情面板
  - ✏ 编辑：编辑该成员信息
  - 👶 添加子女：为该成员添加子女
  - 👫 添加配偶：为该成员添加配偶信息
  - 🌿 展开后代：展开该成员的所有后代节点
  - 🗑 删除：删除该成员

**展开/折叠控制：**
- 工具栏中可选择「展开到第 N 代」
- 支持一键展开全部 / 折叠全部
- 点击节点上的展开/折叠图标可单独控制

**搜索定位：**
- 工具栏搜索框输入姓名，按回车搜索
- 找到后自动定位到该节点并高亮显示，同时高亮其祖先链

### 5. 导出家谱图

在家谱图页面，点击工具栏的「导出」按钮：

1. 选择导出范围：
   - **全部**：导出完整的家谱树
   - **世系**：导出指定人物及其所有后代
2. 点击「导出 SVG」生成高清矢量图
3. 选择保存位置即可

### 6. 打印家谱

在家谱图页面，点击工具栏的「打印」按钮：

1. 配置打印布局和参数
2. 预览打印效果
3. 点击「打印」发送到打印机

### 7. 设置

进入「设置」页面（左侧导航第三项），包含以下功能板块：

#### 🔐 安全设置
- 修改密码：输入当前密码和新密码即可更改

#### 🎨 显示设置
- **世代配色设置**：自定义每一世节点的配色方案
- **字辈管理**：录入和编辑家族字辈，支持按世代排列

#### 📁 数据存储位置（桌面版专属）
- 查看当前数据存储路径
- **更改位置**：选择新的存储目录，可选择是否迁移现有数据
- **恢复默认**：恢复为默认存储路径（`~/ftree-data/`）
- 修改后需重启应用生效

#### 💾 数据管理
- **导出数据**：导出为 JSON 文件（文件名格式：`ftree_export_YYYY-MM-DD.json`）
- **导入数据**：从 JSON 文件导入（将覆盖现有数据，请先备份）
- **创建备份**：手动创建本地备份
- **清除所有数据**：删除所有家族数据（⚠ 不可恢复，请谨慎操作）

#### ℹ 关于
- 显示软件版本等信息

## 🏗️ 技术架构

### 技术栈

| 层面 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Electron 28 | 跨平台桌面应用容器 |
| 前端框架 | React 18 + TypeScript | 组件化开发，类型安全 |
| 构建工具 | Vite + electron-builder | 快速开发构建 + 应用打包 |
| UI 组件库 | Ant Design 5 | 企业级 UI 组件 |
| 样式方案 | Tailwind CSS | 原子化 CSS 快速布局 |
| 图形渲染 | D3.js v7 | SVG 树形图渲染，支持大数据量 |
| 高清导出 | SVG 原生导出 | 矢量图无损放大 |
| HTTP 服务 | Express.js | 内嵌 HTTP 服务，支持浏览器访问 |
| 数据存储 | JSON 分片文件 | 无第三方数据库依赖 |
| 数据加密 | AES-256-GCM + PBKDF2 | Node.js crypto 内置加密 |

### 数据存储方式

数据存储在用户主目录下的 `ftree-data/` 文件夹（可自定义路径），结构如下：

```
~/ftree-data/
├── config.json          # 加密配置、密码哈希
├── index.enc            # 加密的树结构索引
├── family_meta.enc      # 加密的家族元数据
├── details/             # 加密的详情分片文件
│   ├── chunk_000.enc    # 每个文件存储 500 人的详细信息
│   ├── chunk_001.enc
│   └── ...
└── backups/             # 自动备份目录
    └── ...
```

- **索引文件**常驻内存，启动加载速度 < 1 秒
- **详情数据**按需加载，配合 LRU 缓存（容量 200 条），查询响应 < 50ms
- 写入采用原子操作（先写临时文件再重命名），防止数据损坏

## ❓ 常见问题

### Q: 忘记密码怎么办？
A: 密码用于加密数据文件，目前没有密码找回功能。如果忘记密码，数据将无法解密恢复。建议使用常用且容易记住的密码，并定期导出数据备份（明文 JSON）。

### Q: 数据存储在哪里？
A: 默认存储在用户主目录下的 `ftree-data/` 文件夹。Windows 下通常为 `C:\Users\<用户名>\ftree-data\`。可在设置中更改存储位置。

### Q: 支持多少人的家谱？
A: 理论上支持最多 **50,000 人**。采用分片存储 + 虚拟渲染技术，初始仅渲染可视区域内的节点，同屏控制在 500-2000 个节点以内，保证流畅体验。

### Q: 如何在局域网其他设备上访问？
A: 启动应用后，Express 服务器会在 `17800` 端口提供 HTTP 服务。局域网内其他设备可通过浏览器访问 `http://<你的IP>:17800` 使用完整功能。

### Q: 如何迁移数据到新电脑？
A: 有两种方式：
1. **推荐**：在旧电脑上通过「设置 → 导出数据」导出 JSON 文件，在新电脑上通过「设置 → 导入数据」导入
2. **直接复制**：将 `~/ftree-data/` 整个文件夹复制到新电脑的相同位置

### Q: 浏览器模式和桌面应用有什么区别？
A: 功能基本一致，主要区别：
- 浏览器模式不支持原生文件对话框（导出功能使用浏览器下载）
- 浏览器模式不支持自定义数据存储路径
- 桌面应用的数据保存和文件操作体验更流畅
